1. Seven Rules:	Please read and follow the "how to write great commit messages" article and its seven suggestions.
2. Atomic Commits  only:	Do not mass update - each commit should only contain one chage. It could be several files related to one change but not multiple changes combined into one.
3. Clearly document:	"what" this commit changes (not "how" it changes)
4. Do not imply:	be explicit in your comments.
